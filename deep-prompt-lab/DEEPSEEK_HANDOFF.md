# DeepSeek Handoff

把这个文件当作发给 DeepSeek 的工作单。每次只放一个小任务，边界越清楚越好。

## 复制给 DeepSeek 的模板

```text
你是我的代码生成顾问。请只为一个边界清晰的小任务提供第一版可运行代码，不要擅自扩展需求。

任务：
[在这里写一个明确的小功能]

项目上下文与约束：
[在这里写语言、框架、输入输出、已有函数、不能改动的部分]

验收标准：
[在这里写 3-6 条验收条件]

请按下面格式输出：
1. 用 3-6 句话确认你理解的输入、输出、边界条件和潜在歧义
2. 给出完整代码，优先选择稳定、易读、依赖最少的实现
3. 标出你做了哪些假设
4. 列出 3-5 个我应该交回 Codex 做整合与校验的点

不要编造不存在的项目文件，不要把多个功能混在一起，不要输出与当前任务无关的解释。
```

## 当前推荐实战任务

```text
任务：
为一个深度学习 Prompt 网站增加“研究任务交接卡片”生成功能。输入研究主题、已有材料、当前目标，输出一段可直接交给 Codex 的整合说明。

项目上下文与约束：
- 项目是一个静态网站，核心文件是 index.html
- 现有页面已经有 Prompt 模板、分类、抽屉预览和复制功能
- 不要引入新依赖，不要拆分为多文件
- 尽量复用现有 prompts 数据结构

验收标准：
- 新模板能放进现有 prompts 列表
- 字段名清晰，适合深度学习研究场景
- 文案强调交接信息完整性
- 结果可直接复制给 Codex
```

## 使用规则

1. DeepSeek 只负责给草案，不直接决定项目最终结构。
2. DeepSeek 返回的代码不要直接塞进仓库。
3. 下一步把返回内容粘贴到 `CODEX_INTEGRATION.md` 再交给 Codex。
