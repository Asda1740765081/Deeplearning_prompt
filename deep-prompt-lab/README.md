# DeepPrompt Lab

面向深度学习研究者的中文 Prompt 提示词网站，覆盖论文阅读、论文写作、实验设计、代码研发、研究创新和组会汇报等任务。

## 功能

- 28 个结构化研究 Prompt 模板，覆盖阅读、写作、实验、代码、创新与组会汇报
- 新增 `双模型协作` 模板区，支持 `Codex 执行 + DeepSeek 代码草案` 工作流
- 按关键词、难度、评分和收藏筛选，支持随机推荐与最近使用
- 自定义模板、变量自动识别、JSON 导入导出与浏览器本地保存
- 中英文输出、五星评分和可携带填写内容的分享链接
- 7 个深度学习研究 Agent，带角色边界、交接产物与每周研究闭环
- 研究团队总协调 Prompt，可直接交给 Claude、Codex 或 Copilot 执行
- 提供 `交给 DeepSeek 的代码草案单`、`交回 Codex 的整合指令`、`研究任务交接卡`、`仓库上下文打包`、`Codex + DeepSeek 协作总控`
- 分类浏览与关键词搜索
- 填写研究信息后生成完整提示词
- 一键复制、收藏和深色模式
- 桌面端与移动端响应式布局
- 纯静态网站，无需后端或数据库

## 推荐用法

适合深度学习研究里的代码与项目协作：

1. 在网站中打开 `双模型协作` 分类
2. 先用 `交给 DeepSeek 的代码草案单` 让 DeepSeek 产出函数、脚本或局部实现
3. 再用 `交回 Codex 的整合指令` 把草案交给 Codex，按当前仓库结构落地、校正和验证
4. 对于较大的研究任务，先使用 `Codex + DeepSeek 协作总控` 生成一整套拆解与交接流程

## 本地协作文件

- `DEEPSEEK_HANDOFF.md`：发给 DeepSeek 的小任务模板
- `CODEX_INTEGRATION.md`：把外部草案交回 Codex 的整合模板
- `REAL_LINKUP_WORKFLOW.md`：完整联动流程与第一轮实战建议

## 使用

直接打开 `index.html`，或通过 GitHub Pages 部署为公开网站。

收藏和主题设置仅保存在访问者自己的浏览器中。

## GitHub Pages

在仓库的 `Settings > Pages` 中，将发布来源设置为 `Deploy from a branch`，分支选择 `main`，目录选择 `/ (root)`。
