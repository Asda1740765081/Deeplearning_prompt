# Codex Integration

把 DeepSeek 返回的代码或方案贴到这个文件里，然后让我按当前仓库结构整合。

## 复制给 Codex 的模板

```text
请作为当前仓库的执行工程师，先阅读项目结构，再把下面这份外部代码草案稳妥整合进去。

最终目标：
[在这里写最终要落地的功能]

外部草案：
[把 DeepSeek 返回的代码或方案贴在这里]

整合规则：
- 遵循当前仓库风格
- 优先复用已有结构与命名
- 只改必要文件
- 补最小但足够的验证
- 不要改无关功能

请执行时遵循：
1. 先判断草案哪些部分可直接用，哪些需要重写或丢弃
2. 只在必要文件中改动，遵循仓库现有命名、结构和错误处理方式
3. 若草案与现有实现冲突，以项目实际结构为准
4. 若能运行验证，就补上最小但足够的检查；若不能运行，明确说明原因
5. 输出时给我：改了什么、为什么这样改、剩余风险或后续建议
```

## 当前推荐整合目标

```text
最终目标：
继续增强 DeepPrompt Lab 的双模型协作区，让用户可以更顺手地生成“交给 DeepSeek”和“交回 Codex”的交接文本。
```

## 使用规则

1. Codex 负责进仓库、改文件、校正草案、做验证。
2. 如果草案和项目结构不匹配，以项目现状为准。
3. 每次整合后，把最终可复用的内容沉淀回网站或文档。
