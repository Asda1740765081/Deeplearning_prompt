# Real Linkup Workflow

这套流程是给你现在直接开工用的。

## 最顺手的分工

- Codex：拆解需求、查看仓库、改文件、验证结果、整理成最终可用版本
- DeepSeek：产出函数、脚本、局部实现、替代写法、思路草案

## 一次完整联动怎么跑

1. 在 `DEEPSEEK_HANDOFF.md` 里填一个小任务。
2. 把模板发给 DeepSeek。
3. 收到代码后，不直接使用。
4. 把返回内容粘贴到 `CODEX_INTEGRATION.md`。
5. 回到 Codex，对我说：

```text
读取 CODEX_INTEGRATION.md，把里面的 DeepSeek 草案整合进当前项目。
```

6. 我会负责：
   - 看仓库结构
   - 判断草案是否可靠
   - 改文件并尽量验证
   - 告诉你改了什么和剩余风险

## 推荐任务颗粒度

适合先交给 DeepSeek：

- 一个函数
- 一个正则解析器
- 一个数据处理脚本
- 一个图表生成逻辑
- 一页 PPT 结构生成器

适合直接交给 Codex：

- 改现有页面
- 接进已有项目结构
- 修交互逻辑
- 做风格统一
- 做最小验证
